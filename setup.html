The following Setup Guide is a collection of suggestions and hints to get you started with the NGFB plugin. NGFB works great "right out of the box" and you may decide to keep the default settings as-is, but modifying just a few options for your website will help to maximize your social presence and click-through rate.

  1. Create a Facebook Page, Twitter account, and Google+ account (or Page) for your website / business (if you haven't already).

    You can enter your website's social information on the WordPress Settings > Website / Business Social Accounts page. The social account values are used for Google / SEO, Schema, Open Graph, and other meta tags -- including the publisher (Organization) social profiles markup in schema.org JSON-LD format. These social accounts may also appear in Google Search results to highlight your website / business social pages.

  2. Ask your authors to enter their Facebook, Google+, and Twitter contact information on their profile page.

    Each author's contact information can appear in various meta tags and schema.org JSON-LD markup -- but only if they complete their user profile. All URLs found in the author's contact methods will be listed in the author (Person) social profile markup. The author's "Twitter @username" is used to include a URL to their Twitter profile as well.

    Authors should also take a moment to update their Facebook Follower Settings to enable a Follow button on their shared Facebook content. In this way, authors can publish both private (for friends) and public (for friends and followers) content on their Facebook wall.

  3. Review the WordPress Settings > Social Image Dimensions values.

    If your original images are large enough, you should set the Open Graph image dimensions to 1200x1200px cropped or larger (see each option's help popup for more information on image dimensions). Images that are at least 600px wide will be displayed more prominently on Facebook, and 1200px is required to avoid up-scaling images for retina / high PPI displays — so it's worth taking the time to make sure that all your original / full-size featured images are at least 1200px in width and height, along with setting an Open Graph image dimension of 1200x1200px cropped.

    The default Open Graph image dimensions values are 600x600px cropped. You can reduce these values down to 200x200px, but Facebook will display these images as thumbnails in timelines. The minimum recommended image dimensions value is 600x600px cropped — for best results, use 1200px wide images or larger. Aside from Twitter and Google+, most social sites will use the Open Graph image — 1200x1200px cropped is the most compatible dimension, but if you're primarily concerned with images on Facebook, you could choose 1200x630px cropped instead (this is Facebook's preferred image dimension).

    If your original images are large enough, you should also double the default Twitter Card image dimensions from 600x600px to 1200x1200px, and from 300x300px to 600x600px. By offering larger images, you will allow Twitter to provide higher resolution images for retina / high DPI displays.

  4. Review all highlighted options (outlined in orange).

    NGFB will use information from the first post on index / archive and search webpages. If you define a Default Image ID (or Default Image URL), NGFB can use this image as a fallback in case a Post / Page does not contain a suitable image, and/or force the default image on index / archive and search webpages. See the NGFB > General > Images tab for these and other image related options.

    Additional highlighted options you may want to review include the Default Article Topic and Site Description (under the Site Information tab), Facebook Admin Username(s) and Author Name Format (under the Facebook tab), among several others.

    About the Theme Integration tab...

    The NGFB > Advanced > Theme Integration tab has a particularly important option that you should review. The <head> Attributes Filter Hook option is used to add schema markup to your webpage head element. NGFB will check for a standard <head> element in your theme template, and if found, will suggest an automated modification of your theme's header.php template. If you did not see this notice when installing or updating NGFB — or you dismissed it without modifying you theme template — you should review the help popup for this option and make the necessary changes manually.

  5. Submit a few sample Posts and Pages to the Facebook Debugger to verify your Open Graph meta tags.

    A link to the Facebook Debugger is available under the Validate tab in the Social Settings metabox. Submitting a URL to the Facebook Debugger can also be used to clear Facebook's cache (after clicking the "Fetch new scrape information" button a few times).

  6. Submit a Post / Page to the Pinterest Rich Pin Validator and request approval for Rich Pins.

    Pinterest Rich Pins are an extension of Open Graph meta tags, but the Rich Pin Validator can be a little buggy, so double check any errors with the Facebook Debugger to make sure they are actual / real errors. Once approved, you can zoom a Pin to view its Rich Pin information.

  7. Submit index / archive and post / page URLs to the W3C Markup Validation Service to verify your theme templates.

    Social crawlers expect properly coded HTML markup that conforms to current HTML / XHTML standards. If your webpages contain serious markup errors, social and/or search engine crawlers may be unable to parse your webpage content. Fix and/or report any template markup issues with your theme author.

  8. Review the plugin documentation codex on the support website.
